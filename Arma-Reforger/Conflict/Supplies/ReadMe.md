# Припасы - как это работает

Для работы припасов необходимы:
1. Контейнер с самими припасами
2. Табличка - интерфейс для взаимодействия с ними

Таблички(Sign_SupplyStorage_...) делятся на 6 групп:
1. Для арсенала
2. Для пункта заправки
3. Для строительных припасов
4. Для полевого госпиталя
5. Для склада материалов
6. Для пункта лёгкой техники

Табличка является GenericEntity
В ней есть несколько компонентов, но нас интересует `SCR_ResourceComponent`
По умолчанию у него активны категории *Consumers* и *Generators*

В **Storage Range** задаётся расстояние, на котором будут искаться припасы

### Что происходит, когда в грузовик загружаются припасы?



Функция `FindResourceComponent()` вызывается когда игрок открывает свой инвентарь через кнопку Tab или когда нажимает на кнопку на табличке
При чём в первом случае он видит табличку на большом расстоянии, скорее всего как раз 4 метра

Если навести мышку на иконку припасов, то сработает дважды - появление подсказки на это не влияет

48 вызовов с одной табличкой и одними ящиками на 100 припасов
    Возможно, что игра сканирует местность вокруг игрока, разбивая её на квадраты и в них и ищет припасы

Она ищет все предметы в Vicinity(GameCode -> Inventory -> Debug vicinity - показывается в виде зоны, до куда дотягивается) - зоне Поблизости
    При чём предметы на самом игроке так же находятся - чтобы отобразить их в инвентаре в правой части экрана, так как он может с ними в этот момент так же взаимодействовать
    По какой-то причине некоторые объекты находятся по несколько раз - возможно из-за того, что находятся на пересечении нескольких квадратов, в которых ищет игра

**Идея** - может при открытии таблички явно не искать предметы в инвентаре игрока(и не показывать инвентарь в принципе), а запрашивать именно припасы, которые к этой табличке относятся?
